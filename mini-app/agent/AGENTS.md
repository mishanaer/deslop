## Mini App

- Перед любой работой с интерфейсом прочитай `.deslop/mini-app/COMPONENTS.md` и выбери существующий компонент по назначению.
- Собирай интерфейс только из локальных компонентов `src/components/mini-app`. Не импортируй продуктовые компоненты напрямую из `@deslop/mini-app`, не создавай их аналоги и не подключай другие UI-библиотеки.
- Если нужно понять устройство компонента, читай его исходники в `.deslop/mini-app/source`. Не импортируй код из этой справочной папки.
- Для кнопок, полей, переключателей, строк, панелей, меню, модалок, навигации, обратной связи и загрузки всегда используй соответствующий компонент Mini App, а не сырой HTML-контрол.
- Используй только стили, токены, шрифты и иконки Deslop. Не добавляй хардкод цветов, размеров, отступов, радиусов, теней, локальные SVG, inline styles или Tailwind arbitrary values.
- Если нужного компонента нет, остановись и сообщи, какой компонент нужно добавить в Mini App. Не создавай замену внутри продукта без прямого запроса пользователя.
- Отступать от этих правил можно только по прямому запросу пользователя. Исключение нужно явно назвать в результате работы.
- Перед завершением изменений интерфейса запускай `npm run check:mini-app` и исправляй нарушения, а не отключай проверку.
