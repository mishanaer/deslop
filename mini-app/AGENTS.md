# Mini App

- Для нового экрана или флоу, редизайна и визуального аудита сначала прочитай
  `../deslop-ui/SKILL.md`.
- Собирай интерфейс только из компонентов `src/components/` и экспортов `@deslop/mini-app`. Не создавай локальные аналоги, если нужный компонент уже существует.
- Перед добавлением элемента обязательно найди подходящий компонент и его публичный экспорт. Если компонента не хватает, сначала добавь его в `src/components/` как часть Mini App, затем используй в продукте. Не подключай другие UI-библиотеки.
- Используй только токены, шрифты и иконки `@deslop/primitives` через `src/styles/tailwind.css`. Не добавляй произвольные цвета, шрифты, размеры, отступы, радиусы, тени или локальные SVG.
- Не хардкодь визуальные значения в CSS, Tailwind arbitrary values, inline styles или JSX. Новое значение сначала должно стать токеном Primitives.
- Не добавляй новые CSS Modules и локальные стили компонентов. Существующие `*.module.css` — только переходный слой, который нельзя расширять.
- Не добавляй отключения линтеров и проверок. Не меняй `scripts/check-styling.mjs`, `scripts/legacy-css-modules.txt`, конфигурацию линтеров или эти правила ради прохождения проверки; исправляй исходный код. Такие изменения допустимы только по прямому запросу пользователя.
- Отступить от этих правил можно только по прямому запросу пользователя. В таком случае явно назови исключение в результате работы.
- Для Telegram используй только `@lib/twa`. Telegram-функции должны безопасно работать в обычном браузере.
- Перед завершением изменений в коде запусти из корня репозитория `npm run verify`. Не завершай задачу, если проверка не запускалась или завершилась ошибкой.
